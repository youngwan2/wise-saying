## 👁‍🗨 프로젝트 개요
- 약 800개가 넘는 명언 목록을 조회하고, 자신만의 명언 카드를 커스텀하여 공유할 수 있는 차분한 앱 디자인 느낌의 웹 사이트

## 🎫 프로젝트 목적과 방향성
- [목적] 국내, 국외에도 많은 명언 웹이나 앱이 존재 합니다. 하지만, 너무 번잡하고 광범위한 정보를 다루려다 보니 마음에 드는 명언을 곱심으며 감상하기에는 사용자의 시선을 많이 분산시킵니다. 따라서 저는 마음에 드는 명언을 선택하여 차분한 마음으로 집중할 수 있는 환경을 만들고자 했습니다.
- [방향성] 현재 사이트의 목적은 다양한 명언, 속담 정보를 공유하고 가치 있는 정보를 창출하여 향후에는 이윤을 창출 할 수 있는 서비스로 확장해 나가는 것도 있지만, 제일 중요한 것은 제가 배운 모든 것을 하나하나 적용해 나가며 실제 프로덕션에서 운영이 가능한 자격을 갖춘 사이트를 만들어 나가는 것 입니다. 서툰 것도 많고, 부족한 것도 많지만 이 모든 것은 결국 하다보면 해낼 수 있다는 것을 알기 때문에 프로젝트의 개발기간은 따로 정해두지 않았습니다.

※ 이용시 참고사항(24/4/16 기준)
- 사이트 접속은 가능하나 모바일 브라우저의 경우 크로스 브라우징 문제 등으로 TTS 기능에 한하여 동작하지 않을 수 있습니다.
- 구글 로그인의 경우에는 실제 프로덕션에서 사용 시 별도의 확인 절차가 있어서 현재 콜백 URL은 로컬로 자동 지정되어 동작하지 않습니다
- 4월 16일 기준으로 인용문에 대한 위인에 대한 추가적인 정보를 게시하기 위해 데이터를 수집 중에 있습니다. 크롤링에는 법적인 한계가 있어서 직접 데이터를 수집하고 있기에 반영하기에 시간이 많이 걸릴 듯 합니다.

### 배포 및 이미지 처리
|      사용 스텍       | 선택 이유                                                                                                                                              |
| :------------------: | :----------------------------------------------------------------------------------------------------------------------------------------------------- |
|   AWS EC2   | (프로젝트 배포) 로컬 환경과 유사한 환경에서 사이트를 안정적으로 호스팅하기 위해 EC2 사용 |
|   Putty   | (SSH 클라이언트) 윈도우 환경에서 아마존 리눅스 환경으로 빌드 파일 전송 및 배포를 위해 사용 |
|  Nginx      | (TLS) HTTP 패킷 노출 방지 등을 통한 데이터 유출 방지를 위한 목적으로 HTTPS 적용을 위해 활용. 또한 리버스 프록시 기능을 통한 클라이언트와 서버 간의 중간 계층으로서 서버가 그대로 노출되는 문제를 사전에 방지하여 보안상 이점을 누리기 위한 목적도 포함. | 
| AWS S3| 사용자가 명언을 커스텀하고 등록 요청 시 해당 이미지 파일을 사용자별로 S3 버킷에 저장하기 위해 사용, 서버의 데이터베이스 입출력에 대한 오버헤드를 줄일 수 있고, 데이터베이스 입장에서는 해당 이미지에 대한 URL 정보만 정해두고 GET 요청 시 보내주기만 하면 되므로 성능상 이점도 크다는 점이 좋아서 활용. |

- ✨ 현재 배포는 전체 수동으로 이루어지기 때문에 CI/CD 구축은 되어 있지 않습니다. 향후 개발 및 배포의 편의성을 변경될 수도 있습니다.
